AI 에이전트(Claude Code, Codex, 사내 파이프라인…)를 쓰다 보면 HTML 리포트·발표자료·Markdown 문서가 디스크 곳곳에 — 그리고 원격 서버에도 — 쌓입니다. `output_final_v3.html` 같은 이름의 파일이 수백 개가 되면, 정작 필요한 문서를 다시 찾는 일이 고역이 됩니다.

**[Artiview](https://github.com/anears/Artiview)**는 그 폴더들을 탐색 가능한 라이브러리로 바꿔주는 데스크톱 앱입니다. 라이브 렌더 썸네일, 본문 전문 검색, 즐겨찾기·태그, 그리고 로컬 폴더와 똑같이 동작하는 SSH/SFTP 원격 폴더까지 지원합니다. MIT 라이선스 오픈소스로 공개했습니다.

## 앱을 관통하는 세 가지 약속

1. **원본 파일을 절대 수정·삭제하지 않습니다.** Artiview는 읽고 색인만 합니다. 앱 안의 모든 "제거"는 색인 항목만 지웁니다.
2. **모든 데이터는 로컬에 있습니다.** 색인은 내 컴퓨터의 SQLite 파일 하나이고, SSH 비밀번호는 디스크에 기록되지 않습니다.
3. **신뢰할 수 없는 문서는 샌드박스에서 렌더링됩니다.** 에이전트가 만든 HTML은 `allow-same-origin` 없는 iframe에서 열리므로, 악성 문서가 앱이나 시스템에 접근할 수 없습니다.

에이전트 산출물이라는 소재의 특성상 ③이 특히 중요했습니다 — 내가 직접 쓰지 않은 HTML을 매일 수백 장씩 렌더링하는 앱이니까요.

## 주요 기능

- **라이브 썸네일** — 그리드의 각 카드는 실제 문서를 썸네일 크기로 라이브 렌더링한 것입니다. 오래된 스크린샷이 아닙니다.
- **두 겹의 검색** — 라이브러리 검색은 파일명뿐 아니라 **모든 문서의 본문 텍스트**(SQLite FTS5)를 뒤집니다. 파일명이 `output_final_v3.html`이어도 본문에 "매출"이 있으면 찾아냅니다. 뷰어 안에서는 `⌘/Ctrl+F` 문서 내 검색이 하이라이트와 함께 동작합니다.
- **자동 감시** — 등록한 로컬 폴더는 네이티브 파일 워처가 감시합니다. 에이전트가 새 파일을 만들면 몇 초 안에 라이브러리에 나타납니다. 새로고침 버튼이 필요 없습니다.
- **SSH/SFTP 원격 폴더** — 에이전트가 원격 머신에서 돌고 있다면 그 결과물 폴더를 SSH로 등록해 로컬처럼 씁니다. 검색·썸네일·태그가 동일하게 동작하고, 원격 HTML 안의 이미지·CSS는 SFTP로 스트리밍됩니다. 인증은 ssh-agent → ssh-config → 기본 키 순으로 자동 시도합니다.
- **관대한 파일 소실 처리** — 파일이 이동·삭제되거나 드라이브가 뽑혀도 성급하게 지우지 않습니다. 배지로 표시만 하고 태그·즐겨찾기를 보존했다가, 복구되면 원래대로 돌아옵니다.
- **HTML·Markdown·PDF·이미지·텍스트** 를 지원하고, Markdown은 GitHub 스타일 + 코드 하이라이트로 렌더됩니다.

## 스택

**Tauri 2** (Rust 백엔드 + React 프론트엔드) 기반입니다. 색인·전문검색은 SQLite + FTS5, 파일 감시는 네이티브 워처의 디바운스 증분 재스캔, 원격은 SSH/SFTP 연결 풀을 커스텀 `remote://` 프로토콜로 프록시하는 구조입니다. CI가 모든 PR에서 macOS/Linux/Windows 번들을 빌드하고, 태그를 푸시하면 릴리스가 만들어집니다.
